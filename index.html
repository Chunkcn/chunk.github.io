Date Structures & Algorithms Overview

Abstract ：In this blog, I sum some key knowledge about DA&AL.

Basic Concept

  1. Interface and Implementation
  2. Attribute and Entity: an entity is that which contain attributes or properties which may be assigned values.

Algorithms

  1. Five basic operations : Search, Sort, Insert, Update, Delete

Algorithm Analysis:

  1. A Prior Analysis: a theoretical analysis of an algorithm. Before implement , prediction.

  2. A Posterior Analysis: an empirical analysis of an algorithm. After implement.

Algorithm Complexity:

  1. Time Factor: the number of key operations ->Time Complexity

  2. Space Factor: maximum memory space required

    Fixed part and Variable part (depends on the size of the problem)

Asymptotic Analysis:

  1. \(O\) Notation: Upper bound of an algorithm

  2. \(\Omega\) Notation: Lower bound of an algorithm

  3. \(\Theta\) Notation: Both the lower bound and upper bound

Basic Algorithms:

  1. Divide and Conquer

    Merge Sort, Quick Sort, Binary Search, Strassen’s Matrix Multiplication, closest pair

  2. Greedy Algorithm

    The closest solution that seems to provide an optimum solution is chosen.

  3. Dynamic Programming

    An optimal solution can be achieved by using an optimum solution of smaller sub-problems.

Data Structures

Data Definition:

Atomic, Traceable, Accurate, Clear and Concise

  1. Built-in Data Type : Integers, Boolean, Floating, Character and Strings.

  2. Derived Data Type: List, Array, Stack, Queue

Basic Operations

Traversing, Searching, Insertion, Deletion, Sorting, Merging

1. Arrays:

Element and Index

2. Linked List

A sequence of data structures, which are connected together via links.

Link, Next, Linked-List (Header)

  • Double Linked List: Link, Next, Prev, Linked-List(first, last)

  • Circular Linked List

3. Stack

A stack is an Abstract Data Type (ADT). Allow operations at one end only. We can only access the top element of a stack. LIFO

push, pop, peek(get top), isFull, isEmpty

Expression Parsing

  1. Infix Notation: operators are used in-between operands.

  2. Prefix Notation: operator is written ahead of operands.

  3. Postfix Notation: operator is written after the operands.

Using stack implement the convert of infix to prefix, and calculate prefix with stack.

4. Queue

One end is always used to insert data (enqueue) and the other is used to remove data (dequeue). FIFO

Search

1. Linear Search

2. Binary Search

3. Interpolation Search

Find a particular item by computing the probe position.

\(mid = Lo + \frac{Hi -Lo}{A[Hi]-A[Lo]}\times (X-A[Lo])\)

4. Hash Table

Hashing is a technique to convert a range of key values into a range of indexes of an array. Hash Method: modulo,

Deal with conflict: Linear Probing, search the next empty location.

Search Operation: compute the hash code of the key and locate the element.

Sort

  • Bubble Sort

  • Inserting Sort

  • Selection Sort

  • Merge Sort

  • Shell Sort

  • Quick Sort

Graph Data Structure

vertex(vertices), Edge, Adjacency, Path (a sequence of edges)

Basic Operations: Add Vertex; Add Edge; Display Vertex;

Depth First Traversal

use stack

  1. visit the adjacent unvisited vertex. Mark it as visited. Display it and Push it in a stack.
  2. If no adjacent vertex is found, pop up a vertex from the stack. (It will pop up all the vertices from the stack, which do not have adjacent vertices.)
  3. repeat 1. and 2. until the stack is empty.

Breadth First Traversal

use queue

  1. Visit the adjacent unvisited vertex. Mark it as visited. Display it and insert it in a queue.
  2. If no adjacent vertex is found, remove the first vertex from the queue.
  3. Repeat 1. and 2. until the queue is empty.

Tree

Binary Tree: A binary tree has the benefits of both an ordered array and a linked list as search it as quick as in a sorted array and insertion or deletion operation are as fast as in linked list.

Visiting: refers to checking the value of a node when control is on the node.

Key: represents a value of a node based on which a search operation is to be carried out of a node.

Traversing: means passing through nodes in a specific order.

Binary Search Tree Representation

A node’s left child must have a value less than its parent’s value and the node’s right child mush have a value greater than its parent value.

BST Basic Operations:

Insert, search, preorder traversal, inorder traversal. postorder traversal

In-order Traversal

the left subtree is visited first, then the root and later the right sub-tree.

Pre-order Traversal

the root node is visited first, then the left subtree and finally the right subtree.

Post-order Traversal

the root node is visited last. First we traverse the left subtree, then the right subtree and finally the root node.

AVL Tree: height balancing binary search tree

Balance Factor:

the difference between the height of the left and the right sub-trees.

AVL Rotations:

the aim is converting three elements to a 2-level sub-BST

  • Left Rotation
  • Right Rotation
  • Left-Right Rotation
  • Right-Left Rotation

Spanning Tree

all the vertices covered with minimum possible number of edges.

Minimum Spanning Tree (MST)

Kruskal’s Algorithm
  1. remove all loops and parallel edges from the given graph
  2. sort all edges in their increasing order of weight
  3. add the edge which has the least weightage (avoid loop)
Prim’s Algorithm
  1. remove all loops and parallel edges
  2. choose any arbitrary node as root node
  3. check outgoing edges and select the one with less cost

Heap Data Structures

  • Max Heap: the value of parent is great than that of child.

  • Min Heap

Recursion Basics

Properties: 1. Base criteria; 2. progressive approach.
